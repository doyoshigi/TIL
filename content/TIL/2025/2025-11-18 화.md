## 1. AWS Root User 보호
- AWS Root User 보호의 이해

### 상세 내용
- AWS 계정을 처음 생성하고 로그인하면 모든 AWS 서비스, 리소스에 대한 완전한 액세스 권한을 갖게 된다. 이를 AWS Root User라고 한다.
- 기본적으로 AWS Root User에는 이메일 주소와 암호, 액세스키의 2가지 자격 증명이 연결되어 있다.
- Root User에는 강력한 암호, MFA를 설정하고 액세스 키를 공유하지 말고 사용 중지하거나 삭제하며 관리나 일상 작업을 할 때에는 IAM 사용자를 생성하여 해야한다.

### 배운 점의 중요성
- 모든 권한을 가지고 있는 AWS Root User의 권한은 유출되지 않도록 잘 관리해야한다.

### 추가 학습이 필요한 부분
- 자격 증명의 정확한 의미와 해석

## 2. AWS IAM
- AWS IAM의 이해: IAM User, Group, Policy, Role

### 상세 내용
- IAM User는 AWS와 상호 작용하는 사람 또는 서비스를 의미한다.
- IAM User는 이름과 자격 증명으로 구성된다. IAM 사용자 자격 증명은 영구적인 것으로 간주되므로 관리자가 강제 교체할 때까지 사용자와 함께 유지된다.
- IAM Group은 사용자의 그룹이다. 그룹에 할당된 권한은 그룹에 속한 모든 사용자에게 상속된다. 사용자를 그룹으로 구성하고 가능하면 개별 사용자 대신 그룹에 권한을 할당하는 것을 모범 사례로 권장한다.
- 액세스를 관리하고 AWS 서비스 및 리소스에 대한 권한을 제공하려면 IAM Policy을 생성하여 IAM 자격 증명에 연결한다.
- IAM Role은 AWS 자격 증명에 임시 액세스가 필요한 사람이나 주체가 얻을 수 있는 자격증명이다. 임시 액세스이므로 자격 증명은 만료되며, 프로그래밍 방식으로 수임된다.

### 배운 점의 중요성
- IAM은 AWS의 사용자, 서비스에 대한 신원을 확인하고 리소스 액세스를 제어하는 서비스이다. 따라서 이를 이용하여 안전한 서비스를 구축해야 한다.
