## 1. AWS란?
- AWS는 클라우드 컴퓨팅을 제공하는 서비스이다.

### 상세 내용
- 클라우드 컴퓨팅이란 IT 리소스를 인터넷을 통해 제공하는 것이다.
- 클라우드의 장점은 클라우드 서비스 제공자가 인프라를 관리하기 때문에 사용자는 인프라를 신경 쓰지 않고 나머지 부분에 집중할 수 있다.
- 대부분의 클라우드는 종량제 요금제를 사용한다. 종량제 요금제는 사용자가 IT 리소스를 사용한 만큼 비용을 지불하는 것이다.

### 배운 점의 중요성
- 클라우드는 온프레미스 환경 구축이 어려운 소규모 서비스를 배포하고 운영하는 데에 도움이 된다. 최근 AI를 이용하여 빠르게 서비스를 구축하고 배포해야 하는 상황에서 더 도움이 될 것이다.

### 추가 학습이 필요한 부분
- 온프레미스와 클라우드 방식의 비교
- 온디맨드의 개념

## 2. AWS 글로벌 인프라
- AWS 글로벌 인프라 구조: 리전, 가용 영역, 엣지 로케이션, 리전 엣지 캐시

### 상세 내용
- AWS 서비스들은 전역, 리전, 가용 영역 레벨 중 하나로 배포된다.
- AWS 리전은 전 세계에 위치하며 AWS 서비스들을 호스팅한다. 각 AWS 리전은 독립적이다.
- AWS 리전을 선택하는 기준은 데이터 규정 준수, 지연 시간, 요금, 서비스 가용성이 있다.
- AWS 리전은 가용 영역의 클러스터로 구성되어 있다.
- 가용 영역은 1개 이상의 데이터 센터로 구성되어 있으며 데이터 센터는 중복 전력, 네트워크로 연결되어 있다. 따라서 한 개의 데이터 센터에 장애가 발생할 경우 AZ 내부에 다른 데이터 센터가 동작하여 가용성을 보장하게 된다. 이는 재난 같은 상황에서도 계속 서비스를 유지할 수 있게 해준다.
- 데이터 센터는 지연 시간이 짧은 중복 고속 링크를 사용하여 연결된다.
- 엣지 로케이션은 콘텐츠를 캐싱하는 전 세계의 거점이다.
- Amazon CloudFront는 전 세계의 엣지 로케이션 네트워크를 사용하여 콘텐츠를 전송한다.
- 엔드 유저가 엣지 로케이션에 저장된 콘텐츠를 요청할 경우 가장 짧은 지연시간을 제공하는 엣지 로케이션으로 라우팅 된다.
- 리전 엣지 캐시는 엣지 로케이션과 CloudFront의 오리진 서버 사이에 있는 거점으로 엣지 로케이션에서 엔드 유저가 요청한 콘텐츠를 찾지 못했을 경우 CloudFront 오리진 서버에 요청이 가는 것이 아닌 리전 엣지 캐시에서 콘텐츠를 찾는다. 리전 엣지 캐시는 엣지 로케이션보다 캐시 용량이 크기 때문에 자주 접근되는 데이터를 오랫동안 저장할 수 있다. 리전 엣지 캐시로 인해 CloudFront의 오리진 서버로의 트래픽 부담이 경감된다.

### 배운 점의 중요성
- AWS의 글로벌 인프라 구조를 이해하여 서비스를 배포할 때 효율적으로 배포해야 한다.

### 추가 학습이 필요한 부분
- CDN의 개념
- 배포의 개념
- 백본 네트워크

## 3. AWS와 상호작용
- AWS와 상호작용하는 방법: AWS Management Console, AWS CLI, AWS SDK

###     상세 내용
- AWS의 인프라를 조작하는 것은 모두 API를 통해서 조작된다.
- 조작하는 방법은 크게 AWS Management Console, AWS CLI, AWS SDK가 있다.
- AWS Management Console은 웹 사이트에 로그인하여 마우스로 조작하여 프롬프트만 잘 읽는다면 AWS 자원을 쉽게 구성할 수 있다는 장점이 있지만 자원 생성을 위해 여러 페이지를 확인해야 하고 오타가 발생할 가능성이 있으며, 반복적으로 사용하기 번거롭다는 단점이 있다.
- AWS CLI는 Windows, Linux, macOS에서 다운받아 사용하거나 웹 사이트의 콘솔에서 CloudShell로 접속할 수 있다. CLI로 단어 그대로 명령어를 이용하여 AWS를 조작할 수 있다.
- AWS SDK로는 프로그래밍 환경에서 AWS를 조작할 수 있으며 AWS에서 널리 사용되는 언어의 SDK를 지원한다.

### 배운 점의 중요성
- AWS 서비스 조작은 모두 API를 호출해서 조작하는 구조다. API를 호출하는 방법은 AWS Management Console, AWS CLI, AWS SDK를 사용하는 것이다.

### 추가 학습이 필요한 부분

- AWS CLI에서 CloudShell을 제외한 나머지 환경에서의 키 설정 방법

## 4. AWS 보안, 공통 책임 모델
- AWS에서 유저와 AWS의 보안 책임

###     상세 내용
- AWS에서 보안의 책임은 AWS 서비스의 유형에 따라 달라진다. 서비스의 유형은 인프라 서비스와 추상화 서비스가 있다.
- 인프라 서비스에서는 AWS는 기본 인프라 및 기초 서비스를 관리하며 유저는 유저의 데이터를 암호화, 보호, 관리하고 운영 체제 및 애플리케이션 플랫폼을 제어한다.
- 추상화 서비스에서는 AWS는 서버 측 암호화 및 데이터 보호 외에도 인프라 계층, 운영 체제 및 플랫폼을 운영한다. 유저는 유저 데이터를 관리하고, 데이터를 암호화하고, 네트워크 방화벽 및 백업을 통해 데이터를 보호할 책임이 있다.

### 배운 점의 중요성
- 보안에 대한 책임은 매우 중요하다. AWS가 자동적으로 책임져주는 부분이 아닌 것은 잘 파악하여 보안 설정을 통해 보안을 강화해야 한다.

### 추가 학습이 필요한 부분
- 인프라 서비스 vs 추상화 서비스(서버리스)
